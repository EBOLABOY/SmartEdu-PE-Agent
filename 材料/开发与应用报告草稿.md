# 跃课开发与应用报告草稿

## 一、问题提出

小学体育课堂组织具有高度现场性。教师需要在有限课时内同时完成规则讲解、队伍调度、技能练习、计时计分、安全提醒与课堂评价。传统口头讲解和静态 PPT 难以适应体育课堂的动态节奏，导致学生理解规则慢、教师组织负担重、互动资源制作成本高。

## 二、项目目标

本项目开发“跃课：小学体育课堂互动大屏协同创作智能体”，面向小学体育教师提供一个可对话、可修改、可投屏、可导出的智能工作台，用于生成并持续优化体育课堂互动大屏与配套教案。

## 三、系统设计

系统采用三栏结构：

- 左侧为 AI 对话修改区，支持自然语言输入修改需求。
- 中间为项目结构与手动编辑区，展示并维护统一的 Project State。
- 右侧为互动大屏预览区，实时渲染课堂流程、规则、计时计分、路线图、安全提醒等模块。

系统不让 AI 直接自由生成完整 HTML，而是通过结构化 patch 更新已有课堂项目，再由固定模板渲染可运行页面。

## 四、核心功能

- 主案例默认生成：三年级篮球运球接力挑战课。
- 多轮局部修改：规则、队伍、倒计时、路线、安全提醒、主题。
- 风险校验：拦截非法时间、空队伍、删除安全模块等高风险修改。
- 手动编辑：教师可直接修改关键内容。
- 版本回退：保留修改快照。
- 导出能力：JSON、教案 Markdown、证据包 Markdown、单文件 HTML。
- 运行记录：记录 AI 修改、手动编辑、计分变化、倒计时变化、导出与回退动作。

## 五、应用场景

主应用场景为“三年级篮球运球接力挑战课”。系统为该场景提供：

- 游戏规则说明
- 课堂流程展示
- 接力倒计时
- 四队计分器
- 轮换安排
- 接力路线图
- 安全提醒
- 体育精神提示

## 六、应用价值

1. 提升课堂组织效率。教师可快速生成并修改互动资源。
2. 提升学生规则理解效率。大屏把路线、规则和时间可视化。
3. 提升资源复用价值。课堂项目可导出并作为后续模板继续使用。
4. 提升体育课堂的数字化表达质量。系统不是静态课件，而是可运行的课堂工具。

## 七、当前阶段说明

当前版本为 MVP，重点验证产品逻辑和课堂适配性：

- 已实现结构化 Project State 驱动。
- 已实现模拟 AI patch 协同修改。
- 已新增本地轻后端代理，可调用 OpenAI 兼容模型返回结构化 patch。
- 前端采用双层校验：后端校验 JSON 结构，前端校验业务约束。
- 暂未引入账号体系、数据库和教研组共享能力。

## 八、后续优化方向

- 继续优化真实模型提示词与 patch 白名单控制。
- 扩展更多体育项目模板。
- 增加手机遥控端。
- 支持课堂应用记录的自动汇总与反思生成。
- 建立校本体育资源模板库。
