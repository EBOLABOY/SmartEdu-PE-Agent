# 动屏智创演示视频脚本

## 1. 问题引入

镜头展示小学体育课堂常见问题：

- 规则讲解靠口头重复，学生理解慢。
- 计时、计分、轮换都要教师手动兼顾，组织负担重。
- 想做动态大屏，但每节课临时制作成本高。

旁白：

> 小学体育课堂并不缺资源，缺的是可以快速生成、反复修改、直接投屏使用的互动工具。

## 2. 系统亮相

镜头切到系统首页，展示三栏结构：

- 左侧：AI 对话修改区。
- 中间：项目结构与手动编辑区。
- 右侧：互动大屏预览区。

旁白：

> 动屏智创聚焦小学体育课堂组织，通过多轮对话协同生成教案与互动大屏。

## 3. 初稿生成

展示默认主案例：三年级篮球运球接力挑战课。

强调默认生成内容：

- 教学目标
- 课堂流程
- 游戏规则
- 接力倒计时
- 小组积分
- 轮换安排
- 路线图
- 安全提醒
- 体育精神

## 4. 多轮修改演示

依次点击或输入以下指令：

1. 把规则改得更适合三年级学生。
2. 增加四组计分，每组10人，队名用动物名称。
3. 把接力挑战倒计时改成8分钟。
4. 路线图增加右侧返回，避免碰撞。
5. 加入安全提醒，强调掉球后先观察再捡球。
6. 改成高对比投屏风格。

每一步都展示：

- 智能体回复
- 当前模式为“真实 AI”
- 本地代理返回结构化 patch
- 校验提示
- 大屏实时变化
- 中间结构同步更新

旁白重点：

> 系统不是重写全部内容，而是由真实模型先生成结构化 patch，再由前端校验并应用到已有课堂项目。

## 5. 风险校验演示

输入高风险或非法指令：

- 删除安全提醒模块
- 把倒计时改成0分钟
- 清空队伍

展示系统如何拦截，并给出错误提示。

旁白：

> 动屏智创不是无条件接受所有生成结果，而是对体育课堂关键内容进行校验，避免破坏课堂安全和组织逻辑。

## 6. 手动编辑与回退

展示：

- 直接修改队名
- 给某队加分
- 调整倒计时
- 回退上一版本

旁白：

> 教师始终保留最终控制权，AI 只负责协同，不替代教师决策。

## 7. 导出与应用证据

展示导出按钮：

- 导出 JSON
- 导出教案
- 导出应用证据包
- 导出 HTML

再展示导出的 HTML 单独打开并投屏运行。

## 8. 结束语

旁白：

> 动屏智创面向小学体育课堂真实问题，把一次生成升级为多轮协同创作，把静态课件升级为课堂可运行大屏，把单节课工具升级为可沉淀、可复用的体育教学智能体。
